---
description: Use when bucketing an applicant against the operator's rubric.
---

Read the role and rubric from read_yours. Put the applicant in exactly one bucket:

- worth_your_time: clearly matches "what a yes looks like".
- maybe: promising but missing one signal named in the rubric.
- pass: clearly not a fit. A pass is still a kind, honest no later.

Write one plain sentence saying why. No hedging, no jargon. Judge the work, not the
person. If there is no signal at all, that is a pass.
